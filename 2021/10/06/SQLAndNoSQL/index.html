資料模型概述

發表於 2021-10-06 | 分類於 Database

資料模型與查詢語言

  • 資料模型
    • Relational
    • Document
    • Graph

資料庫類別

類別 模型 簡介 特性 使用時機
SQL 關聯式模型 關聯式資料庫(Relational Database)

以其中關係(Table) 被組織起來，每個關係都是元組(tuples，指的是資料列row)的無序組合		 1. 關聯式資料庫，兩個以上的資料表可以產生關聯性。

2. 建立的資料表，欄位就不能變動。

3. 可以複雜的查詢。

4. 儲存重要資料，例如會員資料、訂單資料、交易資料等。

5. 擴充能力弱、成本高。		 1. 資料與資料之間有關聯性.

2. 需要複雜的查詢
NoSQL 非關聯式模型 文件資料庫（Document Database）以文件的形式儲存。
每個文件都是自包含的資料單元，是一系列資料項的集合。
每個資料項都有一個名稱與對應的值。

圖形資料庫（Graph Database) 將資料以圖的方式儲存。
實體會被作為頂點，而實體之間的關係則會被作為邊。
比如我們有三個實體，Steve Jobs、Apple和Next，則會有兩個“Founded by”的邊將Apple和Next連線到Steve Jobs。		 1. 非關聯式資料庫，處理速度快。

2. 可依需求隨時增加或減少，橫向擴充能力強、成本低

3. 儲存單純、無關聯的。

4. 超大規模資料儲存，抓取局部性資料強，更具動態性和表達力		 1. 資料量大

2. 單純的資料、無關聯

3. 在不清楚資料結構的時候
